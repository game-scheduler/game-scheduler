# Task Checklist: Game Clone Feature

## Overview

Allow hosts (and bot managers/maintainers) to clone an existing game session with optional participant carry-over and deadline-based auto-drop confirmation.

## Objectives

- Host can clone a game from any status via `POST /api/games/{game_id}/clone`
- Players and waitlist can be carried over in original order (`YES`, `NO`, or `YES_WITH_DEADLINE`)
- `YES_WITH_DEADLINE` sends confirmation DMs; participants who miss the deadline are auto-dropped
- All existing game mechanics (waitlist promotion, cap enforcement, reminders) work unchanged on cloned game
- No permission system changes — `can_manage_game` already covers host + bot manager + maintainer
